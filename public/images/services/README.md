# Services Preview Images

This directory contains preview/hero images for each service capability page.

## Required Images

Each capability directory should contain a `hero.png` file (1200×600px recommended):

### ai-engineering/
- **hero.png** - Showcase of AI Engineering capabilities
  - Multi-agent automation dashboard
  - Semantic search interface
  - RAG pipeline visualization
  - Suggested style: Dark UI with code/data visualizations, cyan accents

### data-engineering/
- **hero.png** - Showcase of Data Engineering capabilities
  - Data pipeline architecture diagram
  - Schema normalization interface
  - Multi-system integration dashboard
  - Suggested style: Dark UI with data flow visualizations, blue/purple accents

### full-stack-engineering/
- **hero.png** - Showcase of Full-Stack Engineering capabilities
  - Modern web application interface
  - Component library showcase
  - Full-stack architecture diagram
  - Suggested style: Dark UI with responsive layouts, green/yellow accents

### devops-engineering/
- **hero.png** - Showcase of DevOps Engineering capabilities
  - Infrastructure as Code (Terraform/Kubernetes)
  - CI/CD pipeline visualization
  - Self-service PaaS platform interface
  - Suggested style: Dark UI with deployment flows, orange/gold accents

## Image Specifications

- **Format**: PNG (WebP as alternative for better compression)
- **Dimensions**: 1200×600px minimum (2400×1200px for retina displays)
- **File Size**: < 500KB (optimize with compression)
- **Color Mode**: RGB
- **Design Style**:
  - Glass morphism aesthetic
  - Dark mode (matches RegainFlow brand)
  - Subtle neon accents (matching capability theme colors)
  - High contrast for readability
  - Professional, enterprise-grade look

## Design Tools

- Figma (recommended for UI mockups)
- Excalidraw (for architecture diagrams)
- Canva (for quick designs)
- Photoshop/GIMP (for image editing)

## Fallback

Currently, the preview section will show placeholder images or broken image icons until actual images are added. The section is fully styled and functional, waiting for visual assets.

## Notes

- Images should showcase real or realistic examples of the capability
- Avoid stock photos - use actual UI screenshots or custom designs
- Maintain consistent visual language across all capabilities
- Consider adding subtle animations or hover effects in the future
